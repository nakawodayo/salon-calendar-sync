## Phase 分離方針（案1：目的別ディレクトリ）

本プロジェクトでは、思想の「往還（AI ⇄ 人間）」を維持しつつ、試作と本実装を適切に分離するために、以下の構造を採用します。

---

### ディレクトリ構造
```
prototypes/           # Phase 1: 実験・プロトタイプ
  ui-sketches/        # UIの試作・モック
  mvp/                # 検証用の最小実装
src/                  # Phase 2: 本実装（プロダクション志向）
docs/ai-philosophy/   # 思想・運用ドキュメントとログ
```

---

### 運用原則
- Phase 1（prototypes/）では、速度・探索性を優先。要件のあいまいさを受容する
- Phase 2（src/）では、理解・責任・品質を優先。意図と設計を明文化する
- 往還を前提とし、学びは必ず `docs/ai-philosophy/logs/` に記録する
- Phase 1 → Phase 2 への取り込み時は、意図・要件・設計のトレーサビリティを残す

---

### 移行の手順（プロトタイプ → 本実装）
1. 目的の再確認（プロトタイプから得た学びを1〜3行で明記）
2. 要件の明確化（機能境界・前提・非機能）
3. 設計スケッチ（データ構造/責務/フロー）
4. `src/` へ再実装（プロトタイプはそのまま残して比較可能に）
5. ログ更新（実装で得た追加の学び/次の問い）

---

### 命名とコミット規約（任意推奨）
- プロトタイプコミット: `proto: ...`
- 本実装コミット: `feat|refactor: ...`
- 思想変更・重要な学び: `[philosophy] ...`

---

### Q&A
- Q. プロトタイプから直接コピーしてよい？
  A. 基本は「意図を理解したうえで再実装」。例外は検証コードの一時流用
- Q. プロトタイプは消す？
  A. 基本は残す。比較可能性と学習の資産化のため
