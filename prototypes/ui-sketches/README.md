# UI プロトタイプ

**作成日**: 2025-11-11
**目的**: 要件を詰めるための最小 UI プロトタイプ

---

## 概要

システム設計ドキュメントの画面一覧をもとに、MVP に含まれる画面の UI プロトタイプを作成しました。

## 画面一覧

### お客さん側

1. **ホーム画面** (`customer-home.html`)

   - ユーザー情報の表示
   - 予約リクエスト作成ボタン
   - 予約リクエスト一覧へのリンク
   - 注: 次の予約情報表示は Phase 2 で実装

2. **予約リクエスト作成画面** (`customer-create-request.html`)

   - 日時選択（日付、時間）
   - メニュー選択
   - 顧客名（自動入力、読み取り専用）
   - 連絡手段（任意入力）
   - 送信ボタン

3. **予約リクエスト作成画面（カレンダー UI 版）** (`customer-create-request-calendar.html`)

   - ホットペッパービューティー風のカレンダー UI
   - 日付 × 時間のグリッドで空き状況を一目で確認
   - 予約可能な時間帯（◎）をクリックして選択
   - メニュー選択時に所要時間・終了時刻を自動表示
   - 週単位で前後の週に移動可能

4. **予約リクエスト一覧画面** (`customer-requests.html`)
   - 自分の予約リクエスト一覧
   - 状態表示（リクエスト中、確定済み、拒否）
   - 各リクエストの詳細表示

### 美容師側

4. **予約リクエスト管理画面（一覧）** (`stylist-requests.html`)

   - 承認待ちのリクエスト一覧
   - 状態でフィルタリング（すべて、リクエスト中、承認済み）
   - 日時でソート（実装時に実装）
   - 各リクエストの詳細表示・操作ボタン

5. **予約リクエスト詳細・操作画面** (`stylist-request-detail.html`)

   - リクエストの詳細表示（日時、メニュー、顧客名、連絡手段）
   - 承認ボタン
   - 拒否ボタン
   - FIX ボタン（承認済みの場合、Google Calendar に予定作成）
   - 注: 調整機能は Phase 2 で実装

6. **Google 認証・カレンダー選択画面** (`stylist-auth.html`)
   - Google アカウントでログイン
   - Google Calendar API の認可
   - カレンダー一覧表示（認証後）
   - 使用するカレンダーの選択

## 使用方法

1. `index.html` をブラウザで開く
2. 各画面へのリンクから遷移
3. 要件を確認・検討

## 技術スタック

- **HTML5**: マークアップ
- **Tailwind CSS**: スタイリング（CDN 経由）
- **レスポンシブデザイン**: モバイルファースト（最大幅 428px）

## デザイン方針

- **シンプルで分かりやすい UI**
- **モバイルファースト**: LINE アプリ内で使用するため
- **色分け**: お客さん側（青）、美容師側（緑）
- **状態の可視化**: バッジで状態を明確に表示

## 今後の改善点

1. **カレンダー UI**: 日時選択をカレンダーコンポーネントに改善
2. **空き枠表示**: 空き枠を視覚的に表示
3. **ローディング状態**: 処理中の表示
4. **エラー表示**: エラーメッセージの表示方法
5. **確認ダイアログ**: 承認/拒否時の確認

## 注意事項

- これはプロトタイプであり、実際の実装とは異なる場合があります
- 実際の実装時には、Next.js + Tailwind CSS で実装します
- デザインは要件を詰めるための最小限のものです
